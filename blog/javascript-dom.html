Javascript & the DOM

Relationship of Javascript with HTMl and CSS

The way Javascript interacts with HTML and CSS is really fascinating, HTML is like the core structure of a webpage its your paragraphs, your headings, and all your different containers. CSS is the flesh of a webpage, it's how things look, how they are styled and even how they interact with certain changes (like a hover over a button). Javascript is almost like the consciousness of a webpage, you can target specific aspects of the html and css and then force them to do different things, you can write mathmatical equations, random number generators (RNG is the bane of my exisitence LOL), and even take different inputs like keystrokes, or button presses and get them to fire off a sequence of events that start a predesigned animation.


Control-Flow what is it?

Control-Flow is the order in which things in javascript interact, typically javascript runs from top to bottom, however you using different types of javascript code we can run certain things at anytime we chose. This is like how we have a certain order to how we go about our day however different elements of what we do can change but still happen.


Conditional Statments

Conditional statments are things that will fire off under certain conditions (go figure), examples of this are “if… else” statements, (if) runs a part of the code and checks if that is true or false if it is false (else) will run.






Loops

Loops are chunks of code that run for a certain amount of loops untill a criteria is met (they can run forever but this is not optimal and can crash you pc, with no lasting damage) it has many use cases other than repeating a pice of code, it can be used to count a number of objects in an array (container) and when the max number of objects is hit it can fire of a piece of code.


for (let i = 0; i < 5; i++) {
console.log(i);
}

this line of code would loop through and repeat itself untill it hits the number 5 and then stop.


What is the DOM?

The Document Object Model (or DOM for short) is an interface that allows you to view and edit the different elements of a web page, it allows you to see the html, css and also network related queries. It has a built in terminal that allows you to use java script to edit and select different parts of the web page.




Why would you use the DOM?

You would use it to make quick live changes to your css and html on the go (it is not perminant and will disapear on page refresh) but allows you to play around with finding what works so you can make the changes in you code later. It also allows you to select different events that can happen on the page and target specific parts of those events to react in specific ways. It also outputs console log errors if things on the page are not working.


What are the main components of the DOM?

The main components consist of the html view, the computed css styles, a console to read errors and write javascript. Other elements of the DOM inclue A list of the source files, and general stats on the preformance and memory of the site.


What are DOM Events?

DOM events are different different interactions that can happen on a page. When you click a button an event is created, when you hover over something and it reacts, that is also an event. Writing into a text box is also an event, using the DOM you can listen to these and log them, you can also using javascript execute lines of code when an event is triggered.
